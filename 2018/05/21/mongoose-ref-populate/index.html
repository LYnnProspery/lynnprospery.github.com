Mongoose: 关联数据表以及外键填充 Ref & Populate
Mongodb是非关系型数据库，不具备Mysql中具有的join操作，但是mongoose提供了方便的关联、填充功能，使开发不必要手动编写关联查询代码逻辑

There are no joins in MongoDB but sometimes we still want references to documents in other collections. This is where population comes in.

数据关联（外键引用）
Schema的字段中，可以使用ref属性来引用指向另一个Schema

上官方的Demo

var mongoose = require(‘mongoose’);
var Schema = mongoose.Schema;

var personSchema = Schema({
_id: Schema.Types.ObjectId,
name: String,
age: Number,
stories: [{ type: Schema.Types.ObjectId, ref: ‘Story’ }]
});

var storySchema = Schema({
author: { type: Schema.Types.ObjectId, ref: ‘Person’ },
title: String,
fans: [{ type: Schema.Types.ObjectId, ref: ‘Person’ }]
});

var Story = mongoose.model(‘Story’, storySchema);
var Person = mongoose.model(‘Person’, personSchema);
这段代码的storySchema中的author和fans字段都引用了Person类型,
而storySchema中也引用了Story类型

官网有段:

Note: ObjectId, Number, String, and Buffer are valid for use as refs. However, you should use ObjectId unless you are an advanced user and have a good reason for doing so.

经过一些验证和尝试，此处的ref类型, 指的不是数据中其他想关联的字段的类型，而是_id的类型。目前populate填充只能作用于_id。所以ref关联的type要与所关联数据类型的_id字段类型相对应。
